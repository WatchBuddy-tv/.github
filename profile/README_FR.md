## 🔌 Ecosysteme developpeur: ExampleProvider

WatchBuddy repose sur une architecture **decentralisee et extensible**. Vous n'etes pas limite a un catalogue fixe: vous pouvez creer vos propres fournisseurs de contenu et les integrer facilement.

Le projet [ExampleProvider](https://github.com/WatchBuddy-tv/ExampleProvider) est le blueprint officiel pour les developpeurs:

- **Plug & Play:** ajoutez simplement l'URL de votre fournisseur dans WatchBuddy.
- **Developpement isole:** developpez et testez vos scrapers en local, sans dependances externes.
- **Propulse par [KekikStream](https://github.com/keyiflerolsun/KekikStream):** extraction moderne des URLs media. Base sur l'architecture [KekikStreamAPI](https://github.com/keyiflerolsun/KekikStreamAPI).

> 🚀 **Construisez le votre:** utilisez le template `ExampleProvider` pour connecter vos sites preferes a WatchBuddy avec un minimum de code Python.

## ✨ Fonctionnalites principales

- ⏱️ **Synchronisation parfaite:** lecture synchronisee en temps reel pour tous les participants.
- 🌍 **Support media polyvalent:** prise en charge native des flux M3U/HLS et des URLs video directes.
- 🛡️ **Politique No-Hosting:** nous sommes une couche de synchronisation; nous n'hebergeons, ne stockons ni ne distribuons de media.
- ☁️ **Connectivite cloud:** synchronisation optionnelle du profil (favoris + historique) sur vos appareils.
- 💬 **Interaction integree:** chat en temps reel et reactions pendant les watch parties.
